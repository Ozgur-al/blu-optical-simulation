<objective>
Wire per-ray wavelength through the tracer with custom SPD lookup, wavelength-dependent material properties, spectral grid accumulation in both single-thread and MP paths, and project I/O for all new data fields.

Purpose: This is the simulation engine backbone for the spectral feature. Without it, wavelengths are cosmetic — material interactions ignore wavelength and custom SPDs silently fall back to flat white.

Output: A tracer that produces physically meaningful spectral detector grids when sources have non-white SPDs and materials have wavelength-dependent properties. Verified by tests.
</objective>

    - test_spectral_grid_accumulated_for_non_white_spd: Run simulation with warm_white SPD. Verify grid_spectral is not None and has non-zero values in the detector result.
    - test_spd_profiles_project_io_roundtrip: Save a project with spd_profiles and spectral_material_data, load it back, verify dicts match.
    - test_get_spd_from_project_custom_overrides_builtin: get_spd_from_project("my_led", {"my_led": {...}}) should return the custom profile, not the built-in fallback.
  </behavior>
  <action>
    **1. Project data model** (`core/project_model.py`):
    Add two new fields to `Project`:
    ```python
    # SPD profiles: { name: {"wavelength_nm": [...], "intensity": [...]} }
    spd_profiles: dict[str, dict[str, list[float]]] = field(default_factory=dict)
    # Material spectral data: { optics_name: {"wavelength_nm": [...], "reflectance": [...], "transmittance": [...]} }
    spectral_material_data: dict[str, dict[str, list[float]]] = field(default_factory=dict)
    ```

    **2. Spectral helpers** (`sim/spectral.py`):
    - Add `get_spd_from_project(name, spd_profiles)`: Check `spd_profiles` dict first, then fall back to `get_spd(name)`. Return `(wavelengths_nm, intensity)`.
    - Add `blackbody_spd(cct_K, n_bins=N_SPECTRAL_BINS)`: Planck's law `B(lambda,T) ~ lambda^-5 / (exp(hc/lambda*k*T) - 1)`, normalized to peak=1. Return `(wavelengths_nm, intensity)`.
    - Update `sample_wavelengths()` signature to accept optional `spd_profiles: dict | None = None`. When provided, call `get_spd_from_project(spd_name, spd_profiles)` instead of `get_spd(spd_name)`.

    **3. Wavelength-dependent material lookup in tracer** (`sim/tracer.py`):
    - Update `_bounce_surfaces()` signature to accept `wavelengths: np.ndarray | None = None` and `spectral_material_data: dict | None = None`.
    - Inside `_bounce_surfaces()`, after resolving `mat = self._resolve_optics(surf)`, look up spectral data:
      ```python
      optics_name = surf.optical_properties_name or surf.material_name
      spec = (spectral_material_data or {}).get(optics_name)
      ```
    - If `spec` exists AND `wavelengths` is not None:
      - `ray_wl = wavelengths[hit_idx]`
      - `r_vals = np.interp(ray_wl, np.asarray(spec["wavelength_nm"]), np.asarray(spec["reflectance"]))`
      - `t_vals = np.interp(ray_wl, np.asarray(spec["wavelength_nm"]), np.asarray(spec.get("transmittance", [])))`
      - Use `r_vals` and `t_vals` in place of scalar `mat.reflectance` / `mat.transmittance`.
    - If no spectral data, use scalar values as before (backward compatible).
    - Update `_run_single()` to pass `wavelengths` array and `self.project.spectral_material_data` into `_bounce_surfaces()`.
    - Update `sample_wavelengths()` call in `_run_single()` to pass `spd_profiles=self.project.spd_profiles`.

    **4. MP spectral guard + merge** (`sim/tracer.py`):
    - In `run()`, before routing to `_run_multiprocess()`, check:
      ```python
      has_spectral = any(s.spd != "white" for s in sources)
      if has_spectral and settings.use_multiprocessing:
          import warnings
          warnings.warn("Spectral simulation forcing single-thread mode (MP spectral merge not yet optimized)")
          return self._run_single(sources, progress_callback)
      ```
      This is the guard from STATE.md. Full MP spectral merge is deferred.

    **5. Project I/O** (`io/project_io.py`):
    - In `project_to_dict()`: add `"spd_profiles": project.spd_profiles` and `"spectral_material_data": project.spectral_material_data` to the output dict.
    - In `load_project()`: read with `.get("spd_profiles", {})` and `.get("spectral_material_data", {})`, pass to `Project(...)` constructor.

    **6. Tests** (`tests/test_tracer.py`):
    Write all tests from the `<behavior>` block. Use minimal project setups (1 source, 1 surface, 1 detector). Keep ray counts low (1000-5000) for speed.
