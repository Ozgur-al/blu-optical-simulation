  <action>
    **1. Add BVH functions to sim/accel.py:**

    `build_bvh_flat(surface_aabbs: np.ndarray) -> tuple[np.ndarray, np.ndarray]`:
    - Input: `surface_aabbs` — (N, 6) float64 array where each row is [xmin, xmax, ymin, ymax, zmin, zmax].
    - Output: `(node_bounds, node_meta)`:
      - `node_bounds`: (max_nodes, 6) float64 — AABB bounds per node.
      - `node_meta`: (max_nodes, 3) int32 — [left_child_or_surf_idx, right_child, leaf_flag].
        - `leaf_flag == 1`: leaf node; `left_child_or_surf_idx` = surface index in the original array.
        - `leaf_flag == 0`: internal node; children at `left_child_or_surf_idx` and `right_child`.
    - max_nodes = 2*N - 1.
    - Algorithm: recursive median-split along the longest AABB axis of the contained surfaces. Use a Python stack (iterative) to avoid recursion depth issues. When a node contains 1 surface, it becomes a leaf.
    - This function runs in pure NumPy (no @njit). It runs once per simulation start; O(N log N).

    `compute_surface_aabbs(normals, centers, u_axes, v_axes, half_ws, half_hs) -> np.ndarray`:
    - Helper that computes (N, 6) AABBs from surface geometry arrays.
    - For each rectangle: compute 4 corners = center +/- half_w*u_axis +/- half_h*v_axis, then take min/max per axis.
    - Pure NumPy (no JIT needed — runs once).

    `@njit(cache=True, fastmath=True) intersect_aabb_jit(origin, direction, bbox_min, bbox_max) -> float`:
    - Slab-test AABB intersection for a single ray.
    - Returns t_entry (positive) or np.inf if miss.
    - Used inside BVH traversal.

    `@njit(cache=True, fastmath=True) traverse_bvh_jit(origin, direction, bvh_bounds, bvh_meta, surf_normals, surf_centers, surf_u, surf_v, surf_hw, surf_hh, n_nodes, epsilon) -> tuple[float, int]`:
    - Iterative stack-based traversal (stack = np.empty(64, np.int32)).
    - For each leaf: call scalar plane intersection (inline the math — do NOT call another @njit function from inside this @njit function to avoid compilation complexity; duplicate the 15-line intersection math).
    - Returns (best_t, best_surf_idx). best_surf_idx = -1 if no hit.

    `@njit(cache=True, fastmath=True) traverse_bvh_batch(origins, directions, bvh_bounds, bvh_meta, surf_normals, surf_centers, surf_u, surf_v, surf_hw, surf_hh, n_nodes, epsilon) -> tuple[np.ndarray, np.ndarray]`:
    - Batch wrapper: loops over N rays, calls traverse_bvh_jit per ray.
    - Returns (N,) best_t and (N,) best_surf_idx arrays.

    Add BVH kernels to `warmup_jit_kernels()` — build a 4-surface dummy BVH and run traverse_bvh_batch with 4 rays.

    **2. Integrate BVH into tracer.py bounce loop:**

    In `_run_single()`, before the source loop, after expanding solid_faces:
    - Compute total surface count: `n_all = len(surfaces) + len(solid_faces)`.
    - If `n_all >= 50`, build a combined "all planes" set:
      - Create flat NumPy arrays: `all_normals (N,3)`, `all_centers (N,3)`, `all_u (N,3)`, `all_v (N,3)`, `all_hw (N,)`, `all_hh (N,)`.
      - Populate from `surfaces` list first, then `solid_faces`. Record the boundary index `n_surf = len(surfaces)` so you can tell which type each index refers to.
      - Call `compute_surface_aabbs(...)` and `build_bvh_flat(...)`.
      - Set `use_bvh = True`.
    - If `n_all < 50`, set `use_bvh = False`.

    In the bounce loop, when `use_bvh is True`:
    - Replace the four for-loops over surfaces, detectors, sphere_dets, solid_faces with:
      1. Call `traverse_bvh_batch(active_origins, active_dirs, bvh_bounds, bvh_meta, ...)` to get best_t and best_surf_idx for all flat planes.
      2. Map best_surf_idx back to surface type and object index using the boundary index: idx < n_surf → type=0, obj=idx; idx >= n_surf → type=3, obj=idx-n_surf.
      3. Still loop over detectors and sphere_dets separately (they are few and have different accumulation logic; not worth including in BVH).
      4. Compare BVH result with detector/sphere t-values to find overall closest hit.
    - When `use_bvh is False`: keep the existing brute-force for-loops (no behavior change).

    In `_trace_single_source()` (MP path): apply the same BVH logic.

    **3. Implement adaptive sampling in tracer.py `_run_single()`:**

    Modify the per-source emit loop to support batch-based adaptive stopping.

    Current structure: emit all N rays at once, then bounce loop. Change to:
    - When `settings.adaptive_sampling` is True:
      - Emit rays in batches of `settings.check_interval` (e.g., 1000 at a time).
      - After each batch completes the full bounce loop, compute convergence:
        - Track `batch_flux_sum` and `batch_flux_sq_sum` across batches for each detector.
        - After >= 2 batches: `mean = flux_sum / n_batches`, `var = max(0, flux_sq_sum / n_batches - mean^2)`, `std = sqrt(var)`, `I = 1.96 * std / sqrt(n_batches)`, `cv = I / max(mean, 1e-12)`.
        - If `cv <= settings.convergence_cv_target / 100.0`: this source has converged, break the batch loop.
      - If `convergence_callback` is provided, emit `(src_idx, n_rays_traced, cv_pct)` after each batch.
    - When `settings.adaptive_sampling` is False: emit all N rays at once as before (no behavior change).
