  <action>
    Create `backlight_sim/sim/accel.py` with the following structure:

    1. **Try-import guard** at module top:
       - `try: from numba import njit; _NUMBA_AVAILABLE = True`
       - `except ImportError: _NUMBA_AVAILABLE = False` with a no-op `njit` decorator that handles both `@njit` and `@njit(cache=True, fastmath=True)` calling conventions (check `if len(args) == 1 and callable(args[0])` for bare decorator case).

    2. **`intersect_plane_jit(origins, directions, normal, center, u_axis, v_axis, half_w, half_h, epsilon)`**:
       - Decorated `@njit(cache=True, fastmath=True)`.
       - Takes (N,3) origins, (N,3) directions, and plane geometry as separate float64 arrays/scalars.
       - Explicit scalar for-loop over N rays (not vectorized NumPy — Numba compiles this to native).
       - For each ray: compute denom = dot(d, normal), check abs(denom) > 1e-12, compute t, check t > epsilon, compute local u/v coords, check bounds.
       - Returns (N,) float64 array of t-values (np.inf for misses).
       - IMPORTANT: Do not use tuple arguments — Numba needs primitive types. Pass `half_w` and `half_h` as separate floats instead of `size: tuple`.

    3. **`intersect_sphere_jit(origins, directions, center, radius, epsilon)`**:
       - Same pattern: @njit, explicit loop, returns (N,) t-values.
       - Quadratic formula: a = dot(d,d), b = 2*dot(d,L), c = dot(L,L) - r^2, discriminant check, pick nearest positive t > epsilon.

    4. **`accumulate_grid_jit(grid, iy, ix, weights)`**:
       - `@njit(cache=True)` (NOT fastmath — accumulation must be exact).
       - Replaces `np.add.at(grid, (iy, ix), weights)`.
       - Explicit for-loop: `for k in range(len(weights)): grid[iy[k], ix[k]] += weights[k]`.
       - NOT `parallel=True` — scatter-add has race conditions with prange.

    5. **`accumulate_sphere_jit(grid, i_theta, i_phi, weights)`**:
       - Same pattern as accumulate_grid_jit but for sphere detector grids.

    6. **`warmup_jit_kernels()`**:
       - Calls each JIT function once with small dummy arrays (4 rays) to trigger LLVM compilation.
       - Wrapped in try/except to be safe even if Numba is broken.
       - Returns True if warmup succeeded, False otherwise.
       - If `_NUMBA_AVAILABLE` is False, return False immediately (no-op).

    7. **Wrapper functions** for tracer consumption (non-JIT, handle the tuple→scalar conversion):
       - `intersect_plane(origins, directions, normal, center, u_axis, v_axis, size)` — extracts half_w/half_h from size tuple, calls `intersect_plane_jit`.
       - `intersect_sphere(origins, directions, center, radius)` — calls `intersect_sphere_jit`.
       - These wrappers have the same signature as the existing `_intersect_rays_plane` / `_intersect_rays_sphere` in tracer.py, so the tracer can swap them in as drop-in replacements.

    Then add tests in `test_tracer.py`:
    - `test_jit_intersect_plane_matches_numpy()` — create a plane, cast 100 rays, compare JIT vs NumPy t-values within tolerance.
    - `test_jit_intersect_sphere_matches_numpy()` — same for sphere.
    - `test_jit_accumulate_grid_matches_numpy()` — scatter random hit indices+weights into a grid, compare JIT vs np.add.at.
    - `test_jit_warmup_runs_without_error()` — call warmup_jit_kernels(), assert no exception.
    - `test_simulation_deterministic_with_jit()` — run a full _make_box_scene() simulation, verify results match within tolerance (this will be meaningful after Task 2 wires JIT into tracer).
