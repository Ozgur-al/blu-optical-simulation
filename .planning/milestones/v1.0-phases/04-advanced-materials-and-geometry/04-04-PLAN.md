    2. Add `_draw_farfield_lobe(self, sd, result)` method to `Viewport3D` in `backlight_sim/gui/viewport_3d.py`:
       - Per user decision: solid color-mapped mesh surface where radius is proportional to candela, with a cool-to-warm colormap.
       - Build a sphere-like mesh centered at `sd.center`:
         - For each (theta_i, phi_j) bin in `result.candela_grid`, compute vertex radius = `base_scale * candela_grid[theta_i, phi_j] / peak_candela` (normalize so peak = some visible radius, e.g., `sd.radius * 0.8` or a fixed scale factor).
         - Convert (theta, phi, r) to Cartesian: `x = r*sin(theta)*cos(phi)`, `y = r*sin(theta)*sin(phi)`, `z = r*cos(theta)`, offset by `sd.center`.
         - Build vertex array shape (n_theta * n_phi, 3) and triangulate with quad-split faces between adjacent theta/phi rows.
       - Color each face by its candela value using a cool-to-warm colormap:
         - Normalize candela to [0, 1] range (min-max over the grid).
         - Map to RGBA: cool (blue, 0.2, 0.2, 1.0) at 0.0 through warm (1.0, 0.2, 0.2, 0.8) at 1.0. Intermediate: (0.2, 0.7, 0.2) at 0.25, (1.0, 1.0, 0.2) at 0.5, (1.0, 0.6, 0.2) at 0.75. Use `np.interp` per channel for smooth gradient.
       - Create `gl.GLMeshItem(vertexes=verts, faces=faces, faceColors=colors, smooth=True, drawEdges=False)` with `setGLOptions("translucent")`.
       - Store the mesh item in `self._farfield_lobe_item` so it can be removed on the next refresh or when simulation results change.
       - Add a `clear_farfield_lobe(self)` method that removes and nulls `self._farfield_lobe_item` if it exists.

    3. In `gui/object_tree.py`:
       - Add cylinder and prism items under the "Solid Bodies" category (same group as SolidBox).
       - NOTE: CONTEXT.md mentions "Surfaces" as the tree category, but the existing codebase uses "Solid Bodies" for SolidBox. Using "Solid Bodies" is architecturally correct since these are volumetric refractive objects, not flat surfaces. Cylinder and prism follow the established SolidBox tree pattern.
       - Each cylinder: parent item "{cyl.name}" with children "top_cap", "bottom_cap", "side"
       - Each prism: parent item "{prism.name}" with children "cap_top", "cap_bottom", "side_0" through "side_{n-1}"
       - The `_item_group_and_name` helper already handles 3-level items — cylinder and prism face items use the same pattern (grandparent = "Solid Bodies")

    4. In `gui/main_window.py`:
       - Import and create `BSDFPanel` instance. Add as a new tab in the center tab widget (alongside heatmap, 3D viewport, plots). Wire `bsdf_changed` signal to update the BSDF combo in properties panel.
       - Import and create `FarFieldPanel` instance. Add as a new tab. After simulation completes, check if any sphere detector has `mode == "far_field"` — if so, call `far_field_panel.show_result(sd, result)`.
       - After simulation completes, for each far-field sphere detector with results containing `candela_grid`, call `self._viewport.clear_farfield_lobe()` first, then `self._viewport._draw_farfield_lobe(sd, sph_result)` to render the 3D intensity lobe in the viewport. This should happen in the same post-simulation result handler that updates the heatmap panel.
       - Wire SphereDetector mode: in the SphereDetector property form, add a QComboBox for mode selection ("Near-field", "Far-field"). When mode changes, update `sd.mode`. This should be in properties_panel.py SphereDetectorForm (if it exists) or in the general detector form.
       - Add "Add Cylinder" and "Add Prism" actions to the scene menu (alongside "Add Solid Box"). Create with default parameters: cylinder (center at origin, Z-axis, radius=5, length=10), prism (center at origin, Z-axis, n_sides=6, radius=5, length=10).
       - Wire object tree selection: when a cylinder or prism is selected, show the appropriate form. When a face child is selected, show FaceForm (same as SolidBox faces).
       - Wire add/delete for cylinders and prisms: add to `project.solid_cylinders` / `project.solid_prisms`, refresh tree and viewport.
       - Pass `project` to `BSDFPanel.set_project()` on project load/change.
