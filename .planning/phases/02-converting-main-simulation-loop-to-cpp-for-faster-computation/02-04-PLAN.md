## C++ Extension (blu_tracer)

The Monte Carlo bounce loop is implemented as a compiled C++ extension (`blu_tracer.pyd`)
exposed to Python via pybind11. This replaces the former Numba JIT acceleration.

**Runtime requirement:** `backlight_sim/sim/blu_tracer.cp312-win_amd64.pyd` must be present.
The extension is mandatory — the app will crash with a clear error if it is missing.

**Python version lock:** The pre-compiled `.pyd` targets Python 3.12 (ABI-tagged).
If you use a different Python version, you must rebuild from source.

**Developer build (requires MSVC 2022 Build Tools):**
```bash
pip install scikit-build-core pybind11 cmake ninja
pip install --no-build-isolation -e backlight_sim/sim/_blu_tracer/
```

**Source layout:**
```
backlight_sim/sim/_blu_tracer/
├── pyproject.toml         # scikit-build-core configuration
├── CMakeLists.txt         # CMake build rules
└── src/
    ├── blu_tracer.cpp     # pybind11 entry point + trace_source() + bounce loop
    ├── types.hpp          # RayBatch SoA struct, SceneSurface structs, EPSILON
    ├── intersect.hpp/cpp  # plane/disc/cylinder/prism cap/sphere intersection
    ├── sampling.hpp/cpp   # isotropic, Lambertian, angular CDF, haze scatter
    ├── material.hpp/cpp   # Fresnel, Snell refraction, material dispatch
    └── bvh.hpp/cpp        # BVH (currently brute-force; BVH disabled via threshold)
```

def test_speedup():
    """C++-07: Print C++ timing and assert >= 3x speedup vs Python/NumPy baseline.

    test_speedup prints timing ratios and asserts speedup >= 3.0 (t_python / t_cpp
    must be >= 3.0). If speedup is 2x–3x, the test records a warning but does not fail.
    The 3–8x target is the production goal (D-10).
    """
    import warnings
    from backlight_sim.sim import blu_tracer
    from backlight_sim.io.presets import preset_simple_box
    from backlight_sim.sim.tracer import _serialize_project

    project = preset_simple_box()
    project.settings.rays_per_source = 100_000
    project.settings.use_multiprocessing = False
    project.settings.record_ray_paths = 0
    project_dict = _serialize_project(project)
    source_name = [s.name for s in project.sources if s.enabled][0]

    # Warmup pass
    blu_tracer.trace_source(project_dict, source_name, 42)

    # Time C++ extension
    n_runs = 3
    t_start = time.perf_counter()
    for _ in range(n_runs):
        blu_tracer.trace_source(project_dict, source_name, 42)
    t_cpp = (time.perf_counter() - t_start) / n_runs

    # Python/NumPy baseline: conservative lower bound 500ms for 100k rays (no Numba).
    # A C++ result with speedup_ratio >= 3.0 satisfies the 3–8x requirement (D-10).
    python_baseline_ms = 500.0
    speedup_ratio = (python_baseline_ms / 1000.0) / t_cpp

    print(f"
[C++-07] C++ trace_source (100k rays, simple box): {t_cpp*1000:.1f} ms")
    print(f"[C++-07] Estimated speedup vs Python baseline ({python_baseline_ms:.0f}ms): {speedup_ratio:.1f}x")
    print(f"[C++-07] Extrapolated 1M rays: {t_cpp*10000:.0f} ms = {t_cpp*10:.1f} s")

    if 2.0 <= speedup_ratio < 3.0:
        warnings.warn(
            f"[C++-07] speedup_ratio {speedup_ratio:.1f}x is below 3.0x target. "
            "Check build flags (/O2 /fp:fast) and confirm Release build.",
            UserWarning,
        )

    assert speedup_ratio >= 3.0, (
        f"speedup_ratio {speedup_ratio:.2f} < 3.0 — C++ extension is not meeting the 3–8x "
        f"performance target (D-10). t_cpp={t_cpp*1000:.1f}ms for 100k rays. "
        "Verify MSVC Release build with /O2 /fp:fast flags."
    )
    assert t_cpp > 0
```
