Create directory `backlight_sim/gui/commands/` with five files.

**`__init__.py`**: Re-export all command classes for convenience.

**`base.py`**:
```python
from PySide6.QtGui import QUndoCommand  # NOT QtWidgets — Qt6 location

class ProjectCommand(QUndoCommand):
    """Base for all project-mutating commands."""
    def __init__(self, description: str, project, refresh_fn):
        super().__init__(description)
        self._project = project
        self._refresh = refresh_fn
```

**`source_commands.py`**:
- `AddSourceCommand(project, source, refresh_fn)` — redo: append source to project.sources; undo: remove by name
- `DeleteSourceCommand(project, source, index, refresh_fn)` — redo: remove from list; undo: insert at original index
- `SetSourcePropertyCommand(source, attr, old_val, new_val, refresh_fn)` — generic property set with `id()` and `mergeWith()` for rapid edit merging (merge when same source object + same attribute). Uses `hash((id(self._source), self._attr))` for `id()` method.

**`surface_commands.py`**:
- `AddSurfaceCommand(project, surface, refresh_fn)` — same pattern as source
- `DeleteSurfaceCommand(project, surface, index, refresh_fn)`
- `SetSurfacePropertyCommand(surface, attr, old_val, new_val, refresh_fn)` — with merge support

**`scene_commands.py`** — catch-all for other object types:
- `SetPropertyCommand(target_obj, attr, old_val, new_val, refresh_fn)` — generic property command with merge support. Merge uses `hash((id(self._obj), self._attr))`.
- `AddDetectorCommand(project, detector, refresh_fn)` / `DeleteDetectorCommand(project, detector, index, refresh_fn)`
- `AddSphereDetectorCommand` / `DeleteSphereDetectorCommand` — same pattern for sphere_detectors list
- `AddMaterialCommand(project, name, material, refresh_fn)` / `DeleteMaterialCommand` — dict add/remove
- `AddOpticalPropertiesCommand` / `DeleteOpticalPropertiesCommand` — dict pattern
- `AddSolidBodyCommand(project, body, body_type, refresh_fn)` / `DeleteSolidBodyCommand` — handles box/cylinder/prism via body_type string ("box", "cylinder", "prism") mapping to project.solid_bodies / project.solid_cylinders / project.solid_prisms

All commands:
- Do NOT call `redo()` manually — `QUndoStack.push()` calls it automatically
- The `refresh_fn` callback is called at end of both `redo()` and `undo()` to update the UI
- Use `import copy` for deep-copying numpy arrays in old/new values where needed

CRITICAL: Do NOT use `push()` automatically calls `redo()` anti-pattern — push already calls redo, so the command's redo() does the actual mutation. The constructor should NOT perform the mutation.

  <name>Task 2: Wire QUndoStack into MainWindow</name>
  <files>backlight_sim/gui/main_window.py</files>
  <action>
Integrate the command stack into MainWindow:

1. **Import:** `from PySide6.QtGui import QUndoStack` (NOT QtWidgets). Import command classes from `backlight_sim.gui.commands`.

2. **Create stack in `__init__`:** `self._undo_stack = QUndoStack(self)`

3. **Add Edit menu in `_setup_menu()`:**
   - Create `Edit` menu between `File` and `Presets`
   - `undo_action = self._undo_stack.createUndoAction(self, "Undo")` with `QKeySequence.StandardKey.Undo` shortcut
   - `redo_action = self._undo_stack.createRedoAction(self, "Redo")` with `QKeySequence("Ctrl+Y")` shortcut
   - Add both to Edit menu
   - Also add undo/redo actions to the toolbar (after Save, before separator)

4. **Wrap `_add_object()`:** Instead of directly appending/inserting objects, create the appropriate command and push to stack. Example for Sources:
   ```python
   if group == "Sources":
       src = PointSource(f"Source_{n}", ...)
       self._undo_stack.push(AddSourceCommand(self._project, src, self._refresh_all))
   ```
   Same pattern for Surfaces, Detectors, SphereDetectors, Materials, OpticalProperties, SolidBodies (box/cylinder/prism).

   Remove the `self._refresh_all()` call at the end of `_add_object` — the command's `redo()` calls the refresh callback.

5. **Wrap `_delete_object()`:** Create the appropriate Delete command with the object and its index (so undo can re-insert at the correct position). Push to stack.

   For list-based objects (sources, surfaces, detectors, sphere_detectors, solid_bodies, solid_cylinders, solid_prisms): find the index before removal.
   For dict-based objects (materials, optical_properties): store the key and value.

   Remove direct mutation and `self._refresh_all()` call — command handles it.

6. **Property change integration:** The current `_on_properties_changed` just calls `_mark_dirty` and starts the refresh timer. Property changes come from PropertiesPanel signal. To make individual property edits undoable, the PropertiesPanel needs to emit the old and new values along with the target object and attribute name.

   For this plan, use a pragmatic approach: Create a `SetPropertyCommand` that captures a snapshot approach. When `_on_properties_changed` fires, push a generic `SetPropertyCommand` that uses `copy.deepcopy` of the affected object's current state vs its previous state. This avoids modifying every form widget in PropertiesPanel (which is a 1500+ line file).

   Actually, the simpler approach: Store `_last_property_snapshot` (a dict of current property values) when an object is selected. When `_on_properties_changed` fires, compare current values to snapshot, create a SetPropertyCommand for each changed attribute, and update the snapshot. The `mergeWith` on SetPropertyCommand collapses rapid slider drags into one undo step.

   Implementation: In `_on_object_selected`, capture `_last_prop_snapshot = {attr: getattr(obj, attr) for attr in relevant_attrs}`. In `_on_properties_changed`, diff against snapshot, push commands for changes, update snapshot.

   For now, keep it practical: wrap `_add_object` and `_delete_object` with undo commands. For property edits, push a generic `SetPropertyCommand` in `_on_properties_changed` — this gives undo for the most impactful operations (add/delete) immediately, with property undo as a best-effort.

7. **Clear stack on project replace:** In `_new_project()`, `_open_project()`, `_load_preset()`, `_load_variant()`, `_restore_history()` — add `self._undo_stack.clear()` to prevent undo commands referencing stale project objects. This is critical per Pitfall 4 from research.

8. **Mark dirty on undo/redo:** Connect `self._undo_stack.indexChanged.connect(lambda _: self._mark_dirty())` so undoing/redoing marks the project as dirty.
