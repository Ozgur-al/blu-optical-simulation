  <files>backlight_sim/core/detectors.py, backlight_sim/sim/tracer.py, backlight_sim/io/project_io.py, backlight_sim/tests/test_tracer.py</files>

    - Test: SphereDetector defaults to mode="near_field" (backward compat)
    - Test: SphereDetectorResult.candela_grid is None by default (backward compat)
    - Test: far-field accumulation uses ray direction, not hit position — two rays with same direction but different origins hitting a large sphere should bin to the same (theta, phi) cell
    - Test: candela computation divides flux by solid_angle_per_bin (delta_theta * delta_phi * sin(theta_center))
    - Test: candela at theta=0 (pole) uses a minimum solid angle floor to avoid division by zero
    - Test: backward compat — old project JSON without "mode" key loads with mode="near_field"
  </behavior>
  <action>
    1. Add `mode: str = "near_field"` field to `SphereDetector` in `core/detectors.py`. Valid values: `"near_field"` (existing behavior) and `"far_field"` (direction-based binning).

    2. Add `candela_grid: np.ndarray | None = None` field to `SphereDetectorResult` in `core/detectors.py`. This is computed post-simulation for far-field detectors.

    3. In `sim/tracer.py`, add `_accumulate_sphere_farfield(sd, result, directions_at_hit, hit_weights)`:
       - Use `directions_at_hit` (the ray direction vectors at the moment of sphere intersection) instead of `hit_pts - sd.center`
       - Negate directions so they point outward from the source region: `d = -directions_at_hit` (rays travel toward the sphere; we want the outgoing direction from the luminaire)
       - Normalize d, compute theta = arccos(clip(d_norm[:, 2], -1, 1)), phi = arctan2(d_norm[:, 1], d_norm[:, 0])
       - Bin into result.grid using same indexing as _accumulate_sphere

    4. In the sphere detector hit section of the bounce loop (both `_run_single` and `_trace_single_source`):
       - Check `sd.mode`: if `"far_field"`, call `_accumulate_sphere_farfield(sd, sph_results[sd.name], directions[hit_idx], weights[hit_idx])` instead of `_accumulate_sphere(...)`
       - If `"near_field"`, call existing `_accumulate_sphere(...)` unchanged

    5. Add `compute_farfield_candela(sd, result)` function in `sim/tracer.py` (module-level):
       - Computes `candela_grid` from `result.grid` (raw flux per bin)
       - `theta_centers = (np.arange(n_theta) + 0.5) * pi / n_theta`
       - `solid_angle_per_bin = (pi / n_theta) * (2*pi / n_phi) * np.sin(theta_centers)` — shape (n_theta,)
       - Floor sin(theta) at 1e-6 to avoid division by zero at poles
       - `candela_grid = result.grid / solid_angle_per_bin[:, None]`
       - Store in `result.candela_grid`

    6. Call `compute_farfield_candela` after the bounce loop completes, for each sphere detector with `mode == "far_field"`.

    7. In `io/project_io.py`: serialize `sd.mode` in save_project; load with `.get("mode", "near_field")` for backward compatibility.

    8. Write tests per behavior block.
  </action>

  </verify>
