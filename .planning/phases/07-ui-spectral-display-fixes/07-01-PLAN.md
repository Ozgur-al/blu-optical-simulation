**spectral_data_panel.py** — Add a public `update_from_result(self, result)` method to `SpectralDataPanel`. Place it after the existing `_update_chromaticity_for_spd` method (around line 292). This method:
1. Removes any previous `_sim_scatter` item from `_chroma_plot` (separate from the SPD marker `_chroma_scatter`)
2. Finds the first detector with `grid_spectral is not None`
3. Computes per-pixel CIE (x,y) using `spectral_grid_to_xyz` + `xy_per_pixel` from `backlight_sim.sim.spectral`
4. Filters pixels with luminance > 1% of peak (avoids noise clustering at origin)
5. Subsamples to max 2000 points to prevent scatter overload on high-res detectors
6. Plots as a translucent green scatter cloud (`brush=(80, 200, 120, 120)`)
7. Restores fixed CIE diagram view range `[0, 0.85] x [0, 0.92]`

Use `getattr(self, "_sim_scatter", None)` for safe access since the attribute doesn't exist until first call. Keep `_sim_scatter` as a separate item from `_chroma_scatter` (the SPD marker). Wrap the computation in a try/except for resilience — this is a non-critical display enhancement.

Import `numpy as np` at the top of the method if not already imported at module level. Use lazy imports for `spectral_grid_to_xyz`, `spectral_bin_centers`, `xy_per_pixel` from `backlight_sim.sim.spectral` inside the method to avoid circular import risk.
