<task type="auto">
  <name>Task 2: Generate application icon (.ico)</name>
  <files>assets/icon.py, assets/icon.ico</files>
  <action>
Create `assets/` directory.

Create `assets/icon.py` -- a self-contained Python script that generates `assets/icon.ico` using QPainter (PySide6). The icon should:
- Be an optics/light theme icon: stylized converging light rays emanating from a point (like an LED), with a lens or optical element suggested by arcs
- Use teal (#00bcd4) as the primary graphic color on a transparent background
- Render at multiple sizes for ICO format: 16x16, 32x32, 48x48, 64x64, 128x128, 256x256
- Use QPainter to draw: a central bright point (small teal circle/glow), 3-5 light rays fanning outward (teal lines with slight thickness gradient), and a subtle arc suggesting an optical lens element
- Save as `assets/icon.ico` using QImage rendered at each size, then combine into a single .ico file

The script should be runnable standalone: `python assets/icon.py` generates `assets/icon.ico`.

Implementation approach for .ico multi-size:
- For each target size, create a QImage(size, size, Format_ARGB32), fill transparent, paint the icon design scaled to that size
- Use the `PIL` (Pillow) library to combine multiple PNG buffers into a single .ico file. If Pillow is not available, fall back to writing just the 256x256 as a single-size .ico, which is sufficient for PyInstaller.
- Alternatively, use Qt's built-in QIcon.addPixmap for multiple sizes and rely on QPixmap save with ICO format (Qt supports .ico write on Windows).

After creating the script, run it to generate the actual icon.ico file.
  </action>
  <verify>
    <automated>python assets/icon.py && python -c "import os; s = os.path.getsize('assets/icon.ico'); assert s > 1000, f'Icon too small: {s}'; print(f'icon.ico exists, {s} bytes')"</automated>
  </verify>
  <done>assets/icon.ico exists with valid multi-size icon data (>1KB), teal graphic on transparent background, suitable for PyInstaller and Qt setWindowIcon.</done>
